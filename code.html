초심자를 위한 AI 웹앱 만들기 가이드 (Cursor 활용)

이 가이드는 영상 내용을 바탕으로, 코딩 초보자가 AI 도구를 사용해 간단한 '카드 뉴스 웹앱'을 만들고 인터넷에 게시하는 과정을 안내합니다.

영상 속 AI 도구 비교 요약:
  • ChatGPT (Canvas 모델): 코드는 잘 생성하지만, 실행 결과를 바로 볼 수 없고(미리보기 X) 직접 웹사이트로 만들려면 추가 작업이 많이 필요합니다. (요리법만 주는 요리사)
  • Claude: 코드를 생성하고 실행 결과 미리보기를 제공합니다. 하지만 이 역시 바로 웹사이트로 사용하려면 복잡한 개발 환경 설정이 필요합니다. (요리법과 완성된 요리 사진을 보여주는 요리사)
  • Cursor (이 가이드에서 사용할 도구): AI 기반 코드 편집기입니다. 코드 생성, 수정뿐만 아니라, 복잡한 프로젝트 설정, 실행, 웹사이트 게시(배포)까지 도와줍니다. 초보자에게 가장 편리합니다. (요리법 제공, 요리 시연, 실제 요리해서 배포까지 해주는 요리사)

준비물

웹앱을 만들고 게시하기 위해 다음 프로그램 설치 및 계정 생성이 필요합니다 (모두 무료):

  • Cursor 설치: cursor.sh 에서 다운로드 및 설치. AI 기능 사용을 위해 회원가입/로그인이 필요할 수 있습니다.
  • Node.js 설치: nodejs.org 에서 LTS 버전 다운로드 및 설치. (React/Next.js 프로젝트 실행에 필요)
  • Git 설치: git-scm.com 에서 다운로드 및 설치. (코드 버전 관리 및 GitHub 연동에 필요)
  • GitHub 계정: github.com 에서 회원가입. (코드 저장소 역할)
  • Vercel 계정: vercel.com 에서 GitHub 계정으로 회원가입/로그인. (웹사이트 게시 역할)

Step 1: 프로젝트 생성하기 (Cursor 사용)

  1. Cursor 실행: 설치한 Cursor 프로그램을 실행합니다.
  2. 프로젝트 폴더 열기: Cursor 상단 메뉴에서 [File] > [Open Folder...]를 선택하고, 웹앱 프로젝트 파일을 저장할 새 폴더를 선택하거나 만듭니다. (영상에서는 'APP' 폴더를 사용)
  3. Cursor 채팅으로 프로젝트 생성 요청:
    • 단축키 Ctrl + L을 눌러 Cursor 채팅창을 엽니다.
    • 채팅창에 "넥스트 프로젝트를 생성해줘." 라고 입력하고 Enter 키를 누릅니다. (Next.js는 React를 기반으로 웹 개발을 더 쉽게 해주는 인기 프레임워크입니다.)
  4. 프로젝트 생성 명령 실행:
    • Cursor가 Next.js 프로젝트 생성에 필요한 명령어(npx create-next-app...)를 제안합니다.
    • 명령어 블록 옆의 [Run] 버튼을 클릭합니다.
    • Cursor 하단에 터미널 창이 열리면서 설치 과정이 진행됩니다.
  5. 설정 질문 답변하기: 터미널에서 몇 가지 설정 질문을 합니다. 영상에서처럼 아래와 같이 선택하거나, 잘 모를 경우 대부분 Enter(기본값)를 눌러도 괜찮습니다.
    • Would you like to use TypeScript? No (Enter)
    • Would you like to use ESLint? No (Enter)
    • Would you like to use Tailwind CSS? Yes (Enter)
    • Would you like to use `src/` directory? No (Enter)
    • Would you like to use App Router? No (Enter)
    • Would you like to customize the default import alias? No (Enter)

    (선택 후 Enter 키를 누르세요)

  6. 프로젝트 파일들이 자동으로 생성될 때까지 기다립니다. (영상에서는 'card-news-app'이라는 이름으로 생성됩니다.)

Step 2: 기본 코드 확인 및 실행

  1. 프로젝트 폴더 이동 (터미널): Cursor 채팅창이 제안한 대로, 터미널에 프로젝트 폴더로 이동하는 명령어를 실행합니다. (Cursor 채팅창의 [Run] 버튼 클릭) 
    cd card-news-app
  2. 로컬 개발 서버 실행 (터미널): 터미널에 다음 명령어를 실행합니다. (Cursor 채팅창의 [Run] 버튼 클릭) 
    npm run dev
  3. 미리보기 확인: 터미널에 http://localhost:3000 주소가 표시됩니다. Ctrl 키를 누른 상태로 이 주소를 클릭하면 웹 브라우저에서 기본 Next.js 페이지를 볼 수 있습니다.
  4. 기본 페이지 파일 열기: Cursor 왼쪽 파일 탐색기에서 pages 폴더 안의 index.js 파일을 엽니다. 이 파일이 웹사이트의 메인 페이지 내용입니다.
  5. 기본 코드 수정 (연습):
    • index.js 파일 내용 중 return ( ... ); 괄호 안의 코드를 모두 지웁니다.
    • 대신 간단하게 <h1>Hello World</h1> 를 입력합니다. 중요: return 안에는 항상 하나의 최상위 태그(예: <div>...</div>)만 있어야 합니다. 여러 태그를 쓰려면 <div>로 감싸주세요.
    • Ctrl + S를 눌러 파일을 저장합니다.
    • 웹 브라우저의 localhost:3000 페이지가 자동으로 새로고침되며 "Hello World"가 표시되는 것을 확인합니다.

Step 3: AI로 카드 뉴스 앱 코드 생성 및 적용

  1. 기존 코드 삭제: 다시 index.js 파일의 return ( ... ); 안의 내용을 비웁니다.
  2. AI에게 코드 생성 요청:
    • Ctrl + L을 눌러 채팅창을 엽니다.
    • 영상에서처럼 "첨부한 이미지대로 똑같이 디자인 해줘." 라고 입력하고, 미리 캡처해 둔 웹사이트 디자인 이미지를 채팅창에 붙여넣기(Ctrl+V) 합니다. (또는 "카드 뉴스 웹앱 레이아웃을 만들어줘." 라고 텍스트로 요청해도 됩니다.)
  3. 코드 제안 확인: Cursor AI가 카드 뉴스를 위한 코드를 여러 부분으로 나눠 제안합니다. (프로젝트 구조, 필요한 패키지 설치, 기본 구조 설정 등)
  4. 필요한 패키지(라이브러리) 설치:
    • AI가 npm install styled-components 와 같은 패키지 설치 명령어를 제안하면, 먼저 터미널에서 Ctrl + C를 눌러 실행 중인 개발 서버를 중지합니다. (Y 입력 후 Enter)
    • 그 다음, AI가 제안한 패키지 설치 명령어 옆의 [Run] 버튼을 클릭하여 패키지를 설치합니다. (영상에서는 `styled-components` 설치)
  5. 코드 적용하기:
    • AI가 제안한 코드 블록들 (예: `components/Card.js` 파일 생성, `pages/index.js` 파일 수정, `styles/globals.css` 파일 수정) 옆의 [Apply] 버튼을 클릭합니다.
    • Cursor가 해당 파일을 자동으로 생성하거나 수정해줍니다. 변경 내용은 초록색(추가) 또는 빨간색(삭제)으로 표시됩니다.
    • 변경 사항 적용 후에는 반드시 각 파일에서 Ctrl + S를 눌러 저장합니다.
  6. AI에게 추가 요청 (예: 배경 이미지):
    • index.js 파일 내에서 배경을 넣고 싶은 코드 부분을 드래그하여 선택합니다.
    • Ctrl + K를 눌러 선택 영역에 대한 채팅창을 엽니다.
    • Pixabay 등에서 찾은 이미지 주소를 붙여넣고 "이 이미지를 히어로 섹션의 배경이미지로 넣어줘." 라고 요청합니다.
    • AI가 CSS 코드를 수정하여 제안하면 [Apply] 버튼을 눌러 적용하고 저장(Ctrl+S)합니다.
  7. 다시 개발 서버 실행: 터미널에 npm run dev 명령어를 입력하거나 [Run] 버튼을 눌러 개발 서버를 다시 시작하고, localhost:3000 에서 변경된 카드 뉴스 웹앱 디자인을 확인합니다.

Step 4: GitHub에 코드 올리기

만든 코드를 인터넷에 게시하기 위해 먼저 GitHub 저장소에 올려야 합니다.

  1. GitHub 저장소 생성: github.com/new 로 이동하여 새 저장소(Repository)를 만듭니다.
    • Repository name을 입력합니다 (영상에서는 'TodayNews').
    • Public으로 설정합니다.
    • [Create repository] 버튼을 클릭합니다.
  2. GitHub 명령어 복사 및 실행 (터미널): 저장소가 생성된 후 보이는 페이지에서 "...or create a new repository on the command line" 아래의 명령어들을 복사합니다.
  3. Cursor 터미널 (Ctrl + ` 키로 열기/닫기)에 복사한 명령어들을 붙여넣고 Enter 키를 눌러 순서대로 실행합니다. 
    
# 주의: 아래는 예시이며 GitHub에서 제공하는 명령어를 사용하세요.
# git init # 이미 Cursor가 했을 수 있음
# git add README.md # 이 명령어 대신 아래 명령어를 사용하세요!
git add .          # 모든 변경된 파일을 추가
git commit -m "first commit"
git branch -M main
git remote add origin [GitHub에서 복사한 저장소 주소.git]
git push -u origin main

    팁: git add README.md 대신 git add . (점 포함) 명령어를 사용하면 모든 변경된 파일을 한 번에 추가할 수 있습니다. 만약 git push -u origin main 실행 시 오류가 나면 그냥 git push만 입력해보세요.

  4. GitHub 저장소 페이지를 새로고침하여 파일들이 잘 올라갔는지 확인합니다.

Step 5: Vercel로 웹사이트 게시하기

이제 GitHub에 올린 코드를 Vercel을 이용해 실제 웹사이트로 만듭니다.

  1. Vercel 로그인: vercel.com 에 GitHub 계정으로 로그인합니다.
  2. 새 프로젝트 추가: 대시보드에서 [Add New...] > [Project] 버튼을 클릭합니다.
  3. GitHub 저장소 선택: 방금 만든 GitHub 저장소 (예: TodayNews) 옆의 [Import] 버튼을 클릭합니다.
  4. 배포 설정 확인: Vercel이 자동으로 Next.js 프로젝트임을 인식합니다. 특별히 수정할 내용이 없다면 바로 아래의 [Deploy] 버튼을 클릭합니다.
  5. 배포 완료 기다리기: Vercel이 코드를 가져와 빌드(웹 브라우저가 이해할 수 있는 파일로 변환)하고 웹사이트를 게시합니다. 몇 분 정도 소요될 수 있습니다.
  6. 웹사이트 접속: 배포가 완료되면 축하 화면과 함께 미리보기 이미지 아래에 실제 접속 가능한 웹사이트 주소(예: today-news-xxxx.vercel.app)가 표시됩니다. 이 주소를 클릭하여 인터넷에 게시된 웹사이트를 확인합니다!

Step 6: 웹사이트 업데이트하기

웹사이트 내용을 수정하는 것은 매우 간단합니다.

  1. Cursor에서 코드 수정: Cursor에서 index.js 파일 등의 코드를 수정합니다. (예: 부제목 텍스트 변경)
  2. 변경 사항 저장: Ctrl + S로 파일을 저장합니다.
  3. GitHub에 변경 사항 올리기 (터미널): Cursor 터미널에 아래 명령어들을 순서대로 입력합니다. 
    
git add .
git commit -m "내용 업데이트"  # 따옴표 안의 메시지는 자유롭게 작성
git push
  4. 자동 업데이트 확인: 잠시 후 Vercel 대시보드를 확인하거나, 게시된 웹사이트 주소를 새로고침(F5 또는 Ctrl+R)하면 Vercel이 자동으로 GitHub의 변경 사항을 감지하여 웹사이트를 업데이트한 것을 볼 수 있습니다.

결론

축하합니다! Cursor, Next.js, GitHub, Vercel을 이용하여 AI의 도움으로 간단한 웹 애플리케이션을 만들고 인터넷에 게시하는 과정을 완료했습니다. Cursor는 복잡한 설정과 명령어를 상당 부분 자동화해주어 초보자도 쉽게 웹 개발 및 배포를 경험할 수 있게 도와줍니다. 이제 AI와 함께 더 다양한 웹사이트 만들기에 도전해보세요!
